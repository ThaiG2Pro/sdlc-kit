# Output Format Samples & Templates

These files are read on demand by agents to learn the **shape** of each SDLC artifact.
They are NOT auto-loaded into any agent's knowledge base.

## Two kinds of artifact, two sources of truth

**1. OpenSpec-native artifacts — DO NOT format inline; ask OpenSpec.**
`proposal.md`, the spec deltas (`specs/<cap>/spec.md`), `design.md`, and `tasks.md` are owned
by the OpenSpec CLI. Their structure comes from `openspec instructions <artifact> --change "<name>"`
(emits `<template>` + `<instruction>` + a `<rules>` block). The kit's extra conventions
(AC-ID format, ADR ≥2 options, ≥2 checkpoints, …) live in **`openspec/config.yaml → rules:`**
(sourced from `kit/ai/openspec-rules.yaml`, installed by `bin/init.mjs`) — so they show up in
that `<rules>` block automatically. Edit conventions there, in one place, not in agent prompts.
The `*-example.md` files below illustrate the *content depth* OpenSpec's generic template doesn't.

**2. Kit-owned artifacts — use these templates.**
OpenSpec doesn't manage these, so the kit does. Agents reference the template by path and fill it:

| Artifact | Template | Written by |
|----------|----------|------------|
| `qa-report.md` | `qa-report-template.md` | qa (S5) |
| `release.md` | `release-template.md` | developer (S6) |
| `dev-test-report.md` | `dev-test-report-example.md` | developer (S4) |
| `_handoff.md` (CPP) | `handoff-template.md` | all role agents |
| `_state.json` (CPP) | `state-template.json` | all role agents |
| `_decisions.jsonl` (CPP) | `decisions-template.jsonl` | all role agents |
| `_glossary.md` (CPP) | `glossary-template.md` | analyst + architect |
| `_progress.md` | `progress-example.md` | all role agents |
| `openapi.yaml` | `openapi-example.yaml` | architect (S3) |
| `docs/extra-docs/<t>-<s>/intake.md` | `intake-example.md` | intake (pre-S1) |
| `docs/extra-docs/<t>-<s>/ui/<screen>.md` | `ui-screen-example.md` | intake (pre-S1); read by developer (S4) |
| unit test / migration | `unit-test-example.spec.ts` / `migration-example.ts` | developer (S4) |

The CPP templates are shaped to satisfy `cpp-guard` deterministically (handoff "Generated by:"
+ sections, glossary Phase-as-last-column, decisions `type`, enriched `_state.json`). Keep those
structural bits when adapting.

The example **domain is from a reference project** (voucher/commerce) — only the *structure* is
meant to be reused. Replace these with samples from your own project as it matures.
